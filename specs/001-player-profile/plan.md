# Implementation Plan: Player Profile with Rating History

**Branch**: `001-player-profile` | **Date**: 2026-04-06 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-player-profile/spec.md`

## Summary

Add a player profile page accessible from the group leaderboard and match lists. The profile displays a player's Elo rating, win-loss record, recent matches, a rating history chart (SVG line chart derived from confirmed matches), and a head-to-head summary when viewing another player. All data is derived from existing Firestore collections — no schema changes required.

## Technical Context

**Language/Version**: TypeScript 5.6
**Primary Dependencies**: SolidJS 1.9, Firebase 11 (Auth + Firestore), Zod 3.24, @solidjs/router
**Storage**: Firestore — existing collections (`/users/`, `/groups/{id}/members/`, `/groups/{id}/matches/`)
**Testing**: None configured (manual testing via quickstart.md)
**Target Platform**: Web (primary), Mobile via Tauri 2.x, Desktop via Tauri 2.x
**Project Type**: Web application (mobile-first)
**Performance Goals**: Profile page loads in under 2 seconds, chart renders for up to 200 data points
**Constraints**: 320px minimum viewport width, 44px minimum touch targets, no new Firestore collections
**Scale/Scope**: Groups of 20-30 members, ~200 confirmed matches per player maximum

## Phase 0: Research

All technical decisions resolved. See [research.md](./research.md) for details:

1. **Rating history**: Derived by replaying Elo from confirmed matches (no new collection)
2. **Chart**: Hand-rolled SVG line chart (no library dependency)
3. **Navigation**: solid-router route params (`/group/:groupId/player/:playerId`)
4. **H2H**: Client-side filtering of confirmed matches (adequate for small groups)

## Phase 1: Design

### Data Model

See [data-model.md](./data-model.md). Four derived entities:
- **PlayerProfile**: Computed from member + user docs
- **RatingHistoryEntry**: Computed by Elo replay over confirmed matches
- **MatchResult**: Filtered confirmed matches with opponent names
- **HeadToHeadRecord**: Aggregated from matches between two players

No schema changes. No new Firestore collections.

### Service Contract

See [contracts/player-profile.md](./contracts/player-profile.md). Four service functions:
- `getPlayerProfile()` — 2 Firestore reads
- `getRatingHistory()` — 1 query + Elo computation
- `getPlayerMatches()` — 1 query + N user doc reads
- `getHeadToHead()` — 1 query, client-side filter

### Quickstart

See [quickstart.md](./quickstart.md). Five manual test scenarios covering profile viewing, chart rendering, H2H display, match list accuracy, and responsive layout.
